# Core Requirements & Implementation Details

## 1. 材质与光学表现（MeshPhysicalMaterial）
这是实现真实感的灵魂。请为玻璃容器和水体使用 Three.js 的 `MeshPhysicalMaterial`。
- **容器（玻璃）**：要求无色透明。参数设置参考：`transmission: 1`（完全透光）, `opacity: 1`, `ior: 1.5`（玻璃折射率）, `roughness: 0`, `thickness: 0.5`。
- **液体（水）**：嵌套在容器内部，高度约为容器的一半。参数参考：`transmission: 1`, `ior: 1.333`（水折射率）, 配合微弱的蓝色/蓝绿色 `color` 或 `attenuationColor`，并设置合理的 `attenuationDistance` 以体现水体的深度感。
- **背景折射**：为了让 `transmission` 产生肉眼可见的物理折射扭曲，请在 3D 场景中水杯的后方，放置一个带有纹理贴图的平面（Plane），或者设置 Scene 的 background/environment，以便透过水杯能看到扭曲的背景图像。

## 2. 伪物理晃荡动画（Damped Sine Wave）
不使用物理引擎，用纯数学公式控制水面网格的倾斜。
- 监听容器所在 DOM 的鼠标移动（mousemove）事件，计算鼠标移动的水平速度（Velocity X）。
- 当鼠标划过时，将速度映射为水面网格（PlaneGeometry 或 CylinderGeometry 顶面）的**目标倾斜角度（Target Rotation Z）**。
- 在 `requestAnimationFrame` 中，使用**阻尼弹簧算法（Spring Damping）**或**衰减的正弦波（Damped Sine Wave）**，让水面的实际倾斜角向目标角度过渡。
- 视觉结果要求：鼠标快速划过时水面倾斜，随后像真实的半杯水一样左右来回晃荡几下，最终平滑停止。

## 3. 性能优化（个人博客绝对红线）
博客以阅读为主，特效最好不要引起页面卡顿或手机发热。请在代码中严格落实以下优化：
- **IntersectionObserver**：监听该 3D 画布的容器 DOM。只有当它进入视口（isIntersecting === true）时，才触发 `requestAnimationFrame` 渲染循环；离开视口时立刻暂停渲染。
- **像素比限制**：为了防止高分屏移动设备发热，请将渲染器的像素比限制在最高 1.5：`renderer.setPixelRatio(Math.min(window.devicePixelRatio, 1.5))`。
- **抗锯齿**：开启 `antialias: true`，但如果检测到极低端设备可以提供降级方案。
